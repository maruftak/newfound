# Security Policy

## Responsible use

`reconsentry` is built for monitoring assets you **own** or are **explicitly authorized**
to test (for example, domains that are in scope for a bug-bounty or vulnerability
disclosure program). Do not point it at systems you do not have permission to assess.
By default it relies on passive discovery and lightweight HTTP probing.

## Reporting a vulnerability in reconsentry

If you find a security issue in `reconsentry` itself, please report it **privately**:

- Preferred: GitHub → **Security** tab → **Report a vulnerability** (private advisory).
- Do **not** open a public issue for security problems.

We aim to acknowledge reports within 5 business days and to coordinate a fix and
disclosure timeline with you.
